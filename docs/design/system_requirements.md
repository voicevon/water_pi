# Pi Water 系统需求文档 (永久性需求)

## 1. 硬件连接规范
- **液位传感器**：3路开关量输入 (GPIO 17, 27, 22)，带上拉电阻。
- **水泵控制**：3路输出 (GPIO 5, 6, 13)，控制蠕动泵。
- **指示灯**：1路系统心跳灯 (GPIO 26)，每5秒翻转一次。
- **摄像头**：USB 接口，支持按需拍照。

## 2. 采样与抽空业务逻辑
- **前置稳定**：检测到传感器信号 (Logic 0) 后，必须等待 **3分钟 (180秒)** 稳定期方可启动采样。
- **采样阶段**：
  - 固定执行 **3次** 水泵启动循环。
  - **停止间隔公式**：`停止时间 = (配置总时长 - 180) * 0.25` (单位：秒)。
- **信号消失抽空逻辑 (核心需求)**：
  - 在采样流程中/结束后，若传感器信号持续消失满 **3分钟 (180秒)**，系统必须启动 **第 4 次** (抽空) 水泵运行。
  - 抽空运行时长同正常采样时长。
  - **目的**：利用蠕动泵物理特性排空管道，防止不同样本交叉污染。
  - **结束标志**：第 4 次运行结束后，系统方可执行 `reset()` 复位，准备监测下一轮触发。

## 3. MQTT 通讯与远程显示规范
- **远程显示限制**：必须适配 **3行 x 7汉字** 的屏幕规格。
- **日志发布格式**：采用多行字符串形式：
  - **行1**：时间戳 `MM月DD日 HH:mm`。
  - **行2**：通道标识 `通道X:`。
  - **行3**：极简动作内容（严控在 7 汉字内，如 `泵启:第1次`、`关:1次第10s`）。
- **发送频率**：
  - 事件日志：由采样/抽空逻辑驱动触发。
  - 诊断信息 (`info`)：频率固定为 **60秒/次**，确保实时警报带宽。
- **消息持久化**：所有 `publish` 动作必须设置 `retain=True`。

## 4. 部署与维护规范
- **一键部署**：通过 `scripts/deploy_to_pi.ps1` 进行标准打包、同步与服务重启。禁止手动碎片化操作。
- **环境要求**：支持 SSH 免密认证。
